The Problem

An Inaccessible Experience

"How might we redesign the NISRA website so users with visual impairments can read content, navigate pages, and complete sign-ups independently?"
01

Small Text Size

Users with low vision couldn't read program details or form fields, requiring assistance for basic tasks. No option to resize text existed.

02

Poor Color Contrast

Text blended into backgrounds, failing WCAG standards (2.97:1 vs required 4.5:1). Users described it as "straining to read."

03

Unclear Form Labels

Sign-up fields lacked descriptive labels, causing confusion and screen reader incompatibility. Users called it "a guessing game."

Reflection

What I Learned

üí° Key Takeaways

  • Accessibility improvements drive measurable business outcomes
  • Direct user feedback is essential for validating design decisions
  • Iterative testing catches issues early and saves development time
  • Stakeholder buy-in requires connecting design to organizational mission

üîÑ What I'd Do Differently

  • Test with a larger, more diverse sample of visual impairments
  • Partner with accessibility specialists for expert validation
  • Test on multiple screen readers (JAWS, NVDA, VoiceOver)
  • Build accessibility audits into ongoing design workflow

üöÄ Recommended Next Steps

  • Conduct quarterly accessibility audits as site evolves
  • Expand to keyboard navigation and focus indicators
  • Implement user feedback mechanism for accessibility issues
  • Train content creators on maintaining accessibility
